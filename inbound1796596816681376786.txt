1. API / DATABASE PROVIDER
--------------------------------------------------------------------------------

Provider:        Google Firebase (Google Cloud Platform)
Product:         Cloud Firestore (NoSQL document database) — PRIMARY DATA STORE
                 Firebase Authentication — user accounts & login
                 Firebase Cloud Storage — file uploads (photos, documents)
                 Firebase Cloud Functions — server-side HTTP & scheduled jobs
                 Firebase Hosting — web application hosting

Project ID:      canemap-system
Auth domain:     canemap-system.firebaseapp.com
Hosting URL:     https://canemap-system.web.app
Storage bucket:  canemap-system.firebasestorage.app
Firestore region: asia-southeast1
SDK (client):    Firebase JavaScript SDK v12.1.0 (modular)
SDK (server):    firebase-admin (Node.js, Cloud Functions)

Database type:   Document-oriented (NOT relational SQL). There are no SQL tables.
                 Collections = logical tables; Documents = rows; Fields = columns.

Client config:   public/backend/Common/firebase-config.js
Security rules:  firestore.rules, storage.rules, database.rules.json
Indexes:         firestore.indexes.json

Note: Firebase Realtime Database is configured but locked (read/write: false).
      All application data lives in Firestore.

Alternative APIs used by the app (non-database):
  - OpenStreetMap / Leaflet — map display (not a data API for business records)
  - Weather APIs — advisories (referenced in handler dashboard; not primary storage)

--- 3.1 users ---
Path: /users/{userId}
Document ID: Firebase Auth UID (recommended pattern)

| Field                  | Type        | Description |
|------------------------|-------------|-------------|
| uid                    | string      | Firebase Auth UID |
| name / fullname        | string      | Display name |
| email                  | string      | Login email |
| role                   | string      | farmer | handler | driver | sra | admin | system_admin | super_admin | security_admin |
| status                 | string      | pending | verified | (account state) |
| emailVerified          | boolean     | Email verification flag |
| phone (?)              | string      | Contact number |
| photoURL (?)           | string      | Profile image URL |
| createdAt              | timestamp   | Registration time |
| lastLogin (?)          | timestamp   | Last successful login |
| verifiedAt (?)         | timestamp   | When email was verified |
| failedLoginAttempts    | number      | Brute-force tracking |
| lastFailedLogin (?)    | timestamp   | Last failed attempt |
| roleUpdatedAt (?)      | timestamp   | When role last changed (e.g. driver approval) |

Relationships:
  - Linked 1:1 with Firebase Auth user (same UID)
  - Referenced by fieldId ownership (fields.userId), tasks, records, reports, etc.

6. ENTITY RELATIONSHIP SUMMARY

  Firebase Auth (uid)
       |
       +---- users/{uid}  (profile, role)
       |
       +---- fields/{fieldId}  (owned via userId / landowner_id)
       |         +---- remarks/{remarkId}
       |
       +---- records/{recordId}  (userId + fieldId)
       |         +---- bought_items/{itemId}
       |         +---- vehicle_updates/{updateId}
       |
       +---- tasks/{taskId}  (fieldId, handlerId, assignedTo[])
       |
       +---- reports/{reportId}  (handlerId, fieldId)
       |
       +---- field_joins/{joinId}  (userId, handlerId, fieldId)
       |
       +---- Drivers_Badge/{uid}
       |
       +---- notifications (userId or role broadcast)

  field_workers, driver_rentals, task_logs — link users <-> fields


--------------------------------------------------------------------------------
7. COMPOSITE INDEXES (QUERY PERFORMANCE)
--------------------------------------------------------------------------------

Defined in firestore.indexes.json. Key indexes include:

  - fields: status + submittedAt; userId + createdAt; userId + status
  - records: fieldId + recordDate; fieldId + createdAt; userId + createdAt
  - tasks: handlerId + status + completedAt; fieldId + archived; assignedTo + createdAt
  - reports: handlerId + submittedDate; status + submittedDate
  - notifications: userId + timestamp
  - user_activities: userId + timestamp
  - join_fields (collection group): fieldId + status; fieldId + user_uid
  - admin_security_logs: eventType + timestamp
  - driver_rentals: driverId + requestDate
  - Drivers_Badge: open_for_rental + fullname
  - task_logs: fieldId + logged_at

8. HOW TO EXPORT LIVE DATA (FOR TURNOVER PACKAGE)
--------------------------------------------------------------------------------

Schema (this document) describes STRUCTURE. For actual DATA export:

  Option A — Project script (recommended):
    1. Firebase Console → Service accounts → Generate new private key
    2. export GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"
    3. npm run export:system-database
    4. Output folder: "System Database/" (JSON files + auth users list)

  Option B — Firebase Console:
    Firestore → Import/Export → Export to Google Cloud Storage bucket

  Option C — Manual:
    Firebase Console → Firestore → browse collections → export per collection

Include this schema .txt file together with the data export in your turnover ZIP.
